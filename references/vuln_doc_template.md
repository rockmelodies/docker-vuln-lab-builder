# 漏洞靶场说明文档模板

本文档为漏洞靶场标准说明文档的模板，生成靶场文档时按此结构填充内容。

---

## 文档结构

生成的 `VULN_LAB_README.md` 必须包含以下章节：

### 1. 标题与元信息

```markdown
# [漏洞名称] 漏洞靶场

| 属性 | 值 |
|------|------|
| 漏洞类型 | [如: SQL注入 / XSS / RCE / SSRF / 文件上传 / 反序列化] |
| 难度等级 | [初级 / 中级 / 高级] |
| CVE编号 | [如: CVE-2021-44228，无则填 N/A] |
| CVE链接 | [CVE详情页URL] |
| 影响版本 | [受影响的软件版本范围] |
| 镜像名称 | [Docker镜像名] |
| 构建时间 | [YYYY-MM-DD HH:MM:SS] |
```

### 2. 靶场概述

简要描述靶场目的和背景，2-3段文字：
- 靶场模拟的真实场景
- 漏洞的典型危害
- 学习目标

### 3. 环境信息

```markdown
## 环境信息

### 容器配置

| 配置项 | 值 |
|--------|------|
| 镜像名称 | [镜像名:标签] |
| 容器名称 | [容器名] |
| 操作系统 | [容器内OS] |
| 特权模式 | [是/否] |
| 网络模式 | [bridge/host/none] |

### 端口映射与访问地址

| 服务 | 容器端口 | 宿主端口 | 协议 | 访问地址 |
|------|----------|----------|------|----------|
| [服务名] | [端口] | [端口] | [TCP/UDP] | [URL] |

### 启动命令

```bash
# 快速启动
docker run -d -p [端口映射] --name [容器名] [镜像名]

# 使用docker-compose
docker-compose up -d
```
```

### 4. 漏洞描述与原理

详细说明漏洞的技术原理：
- 漏洞成因分析
- 攻击面和攻击路径
- 漏洞触发条件
- 相关代码片段（如适用）

### 5. 漏洞利用步骤

分步骤描述漏洞利用过程：

```markdown
## 漏洞利用步骤

### 步骤1: [步骤名称]

**目标**: [本步骤目标]
**操作**: [具体操作描述]

```bash
# 示例命令或请求
curl http://127.0.0.1:8080/vulnerable_endpoint -d "payload"
```

**预期结果**: [预期看到的结果]

### 步骤2: [步骤名称]
...
```

### 6. 漏洞修复建议

提供修复方案：
- 临时修复措施
- 永久修复方案
- 安全编码建议
- 防御检测方法

### 7. 参考链接

```markdown
## 参考链接

- [CVE详情](https://nvd.nist.gov/vuln/detail/CVE-XXXX-XXXXX)
- [漏洞分析文章](URL)
- [官方安全公告](URL)
- [修复补丁](URL)
```

---

## 文档生成注意事项

1. 所有端口映射和访问地址必须与实际运行容器一致
2. CVE编号和链接必须准确可查
3. 漏洞利用步骤必须可复现
4. 修复建议必须具体可操作
5. 文档使用中文编写
6. 代码块标注正确的语言类型
7. 所有URL必须可访问
